Feature Set

Everything that matters for daily API work

These are implemented now and built for production workflows, not demo checkboxes.

Native speed that stays smooth

Compiled .NET desktop app with Avalonia rendering. Instant startup, low memory use, and stable performance on large responses.

Keyboard-first navigation

Use Ctrl+P to jump to any request, Alt+R to reveal the active request, Ctrl+Enter to send, and Ctrl+S to save context.

Filesystem-native collections

Collections are folders and requests are plain files, so your API work is git-friendly, diffable, and shareable by default.

Dynamic environments

Variables can be static values, generated mock data, or values extracted from previous responses using JSONPath, XPath, or regex.

Secret-safe flows

Secret variables are masked in the UI and not stored in history as plaintext, so auth flows stay practical and safer.

History as a real power tool

Auto-capture request and response snapshots, then filter by status, headers, dates, body text, collection, and request name.

Import without disruption

Bring in Postman, Insomnia, and OpenAPI/Swagger collections as new sets or merge additively into existing projects.

Native Bruno compatibility

Point Callsmith at an existing Bruno folder and work directly with the same .bru files without conversion or lock-in friction.

Full visibility layout

Request editor and response viewer stay visible together, with resolved URL always shown and flexible split layout controls.
